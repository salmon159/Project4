YFS_AUTO_USER_CONSTRAINTS
( TRANSACTION Table )
This table stores transactional constraints automatically created by the system. Transactional constraints are temporary in nature, and are automatically purged by the system when the corresponding transaction data is purged. When tasks are suggested to a user, these constraints have to be met. If no constraints are specified here, it means that any type of task can be suggested to the user.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.
