Entity Definition
YFS_BARCODE_TRANSLATION
( CONFIGURATION Table )
This table defines how barcodes are validated based on the context of the data.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
BARCODE_TRANSLATION_KEYChar (24)The primary key for the YFS_BARCODE_TRANSLATION table.  
ORGANIZATION_CODEChar (24)This is the organization ID for the owner of that bar code.
A logical foreign key to the following table:
  • YFS_ORGANIZATION 
BARCODE_TYPEVarchar (100)This identifies the data type of the field being scanned in the UI. BarCodeDataType attribute in the yfsdatatypes.xml must be set to one of the two values. For an exhaustive listing of supported data types, see Appendix A. All supported bar code types are available under common code of type 'BARCODE_TYPE'. 
TRANSLATION_SEQUENCENumber (9,0)The barcode for a given barcode_type is translated in the sequence specified here. For example, if barcode_data_type 'Item' has three barcode_sources defined called ItemID, UPC and UPCCaseCode. If the seq# were 2,3,1 respectively, then the order of translator call will be UPCCaseCode, ItemID and UPC respectively. 
DESCRIPTIONVarchar (100)Description of the validation setup. 
VARIABLE_LENGTH_FLAGChar (1)Y/N. Used to indicate if the barcode length is not fixed. Ex: scanning a item bar code, itemID could be of different lengths. 
BARCODE_LENGTHNumber (14,0)Length of barcode. Used to identify the barcode when validating the scanned data. This is the length of the entire stream scanned. 
BARCODE_VAL_START_POSITIONNumber (14,0)Bar code start position. 
BARCODE_VAL_END_POSITIONNumber (14,0)Bar code end position. 
BARCODE_TRANSLATION_SOURCEVarchar (100)Identifies a predefined validation source such as ITEM, ITEM_ALIAS, etc. or an externally translated source (EXTERNALLY_TRANSLATED). The externally translated source is implemented using an application extension. A flow must be associated with all externally translated sources.For an exhaustive listing of bar code translation sources, see appendix A. All bar code translation sources supported out of the box are available under common code of type 'BARCODE_TRANSLATION_SOURCE'. 
ALIAS_NAMEVarchar (100)Only valid when the translation source is ITEM_ALIAS. 
FLOW_KEYChar (24)Will define the flow executed when the translation is done externally (EXTERNALLY_TRANSLATED). 
APPLICATION_IDENTIFIERVarchar (100)Any alphanumeric string. If the value is present, then the part of the bar code from the start position up to the length of the application identifier is validated to check that it matches the application identifier. Basically, this allows selection of a data source based on the nature of the data. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications. 
CREATETSDateTimeTimestamp when the record was created. 
MODIFYTSDateTimeTimestamp when the record was last modified. 
CREATEUSERIDVarchar (40)User who created the record. 
MODIFYUSERIDVarchar (40)User who modified the record. 
CREATEPROGIDVarchar (40)Program that created the record. 
MODIFYPROGIDVarchar (40)Program that modified the record.
