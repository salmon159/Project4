Entity Definition
YPM_PRICING_RULE
( MASTER Table )
This table provides the information for each pricing rule.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
PRICING_RULE_KEYChar (24)The primary key for the YPM_PRICING_RULE table.  
DESCRIPTIONVarchar (2000)Description of the rule. 
PRICING_RULE_NAMEVarchar (120)Name of the pricing rule. 
IS_CUSTOM_RULEChar (1)Indicates whether the rule is custom or not. 
PRICING_STATUSVarchar (40)Indicates the status of the pricing rule. The valid values are:
  • ACTIVE: Indicates that pricing entity is active.
  • INACTIVE: Indicates that pricing entity is not active. 
START_DATE_ACTIVEDateTimeDate from which the rule is active. 
END_DATE_ACTIVEDateTimeAfter this date the rule is not active. 
CURRENCYVarchar (20)Currency in which this rule is applicable.
A logical foreign key to the following table:
  • YFS_CURRENCY 
RULE_TYPEVarchar (40)Indicates the type of the rule. The valid values are:
  • COMBINATION: Indicates that this rule is applied based on combination of items in the order.
  • ORDER_TOTAL: Indicates that this rule is applied based on order total before shipping or tax.
  • ITEM_QUANTITY: Indicates that this rule is applied based on item and it's quantity in the order.
  • SURCHARGE: Indicates that the rule is for applying a surcharge.
  • GLOBAL_EXCLUSION: Indicates that the rule is for excluding certain items, categories or attributes from all pricing rules. 
IS_ITEM_RULEChar (1)Indicates whether the rule is targeted on items or not. 
ASSOC_IN_CAT_ATTR_TRIGChar (1)Indicates whether inclusion trigger category and attributes are associated or not. 
ASSOC_IN_CAT_ATTR_TARChar (1)Indicates whether inclusion target category and attributes are associated or not. 
ASSOC_EX_CAT_ATTR_TRIGChar (1)Indicates whether exclusion trigger category and attributes are associated or not. 
ASSOC_EX_CAT_ATTR_TARChar (1)Indicates whether exclusion target category and attributes are associated or not. 
COPY_TRIG_INCLUDES_TO_TARGChar (1)Indicates, Trigger Inclusions are copied to Target Inclusions. Default is N 
COPY_TRIG_EXCLUDES_TO_TARGChar (1)Indicates, Trigger Exclusions are copied to Target Exclusions. Default is Y 
IS_MANUAL_RULEChar (1)Indicates whether this pricing rule is a manually applied adjustment rule or not. A manual pricing rule is suggested by the pricing service based on contents or total of the order, which then again should be specified with the amount of adjustment to be applied against the order. 
IGNORE_RULE_ON_MANUAL_ADJChar (1)Indicates that this rule is not applicable on entire order or lines that have either manual adjustments or manual rule adjustments specified. 
MINIMIZE_NUMBER_OF_SHIPMENTSChar (1)Indicates whether to optimize on the number of shipments or not. Applicable only for shipping category of rules. 
CARRIER_SERVICE_CODEVarchar (40)The carrier service that this rule applies on. Applicable only for shipping category of rules.
A logical foreign key to the following table:
  • YFS_CARRIER_SERVICE 
IS_COUPON_RULEChar (1)Indicates whether this rule is a coupon or not. 
ITEM_OPERATOR_CODEVarchar (40)Indicates how the trigger for the rule is evaluated. Applicable only for combination type of rule. The valid values are:
  • ALL_ITEMS: Indicates that the rule condition is satisfied if all of the items in the condition is present.
  • ANY_ITEM: Indicates that the rule condition is satisfied if any of the item in the condition is present. 
RULE_CATEGORYVarchar (40)Indicates the category to which the rule belongs. The valid values are:
  • PRICING: Indicates that this rule belongs to pricing category. This rule generates adjustments for extended price of a line or for order total before shipping and tax.
  • SHIPPING: Indicates that this rule belongs to shipping category. This rule generates shipping surcharge for a line or adjustment on shipping charge of the order. 
ORGANIZATION_CODEChar (24)Pricing organization to which the configuration belongs.
A logical foreign key to the following table:
  • YFS_ORGANIZATION 
SELLER_ORGANIZATION_CODEChar (24)Seller who maintains this pricing rule. Promotion is defined by the seller.
A logical foreign key to the following table:
  • YFS_ORGANIZATION 
IS_ITEM_ATTRIBUTE_VALUE_RULEChar (1)Indicates that this item targeted rule is defined on item attribute values. Applicable for all types of rules. 
IS_CATEGORY_RULEChar (1)Indicates that this item targeted rule is defined on category values. Applicable for all type of rules. 
DISTRIBUTE_ADJUSTMENTChar (1)Boolean indicating if the rule adjustments should be distributed to lines or not.  
IGNORE_RULE_ON_PRICE_LOCKChar (1)If set indicates that the rule should not be fired if the line item's unit price is locked. The line item can be the target or the rule trigger. Applicable only for rules that are item targeted, like combination and item quantity rules. 
PAYMENT_TYPEVarchar (20)The rule is applicable for this Payment Type. 
EXCLUSIVITY_TYPEVarchar (40)Exclusivity Type of the Pricing Rule The valid values are:
  • ALL: Can combine with any other pricing Rule. (default value)
  • RULE_TYPE_ITEM: Cannot combine with Pricing Rules of same RuleType for same Item.
  • RULE_TYPE_ITEM_INST: Cannot combine with Pricing Rules of same RuleType for same Item instance.
  • RULE_TYPE_ORDER: Cannot combine with Pricing Rules of same RuleType for same Order.
  • ONE_RULE_WITHIN_ID: Cannot combine with any other Pricing rule within the same id
  • GLOBAL: Cannot combine with any other Pricing rule 
EXCLUSIVITY_IDVarchar (100)ExclusivityID is used to group set of Pricing Rules for mutual exclusivity. Exclusivity Type settings are applicable among Pricing Rules having same ExclusvityID 
PRIORITYNumber (10,0)This attribute is used to prioritize Pricing Rule among a list of applicable pricing rules with same Exclusivity settings. Higher the value of this attribute, higher the priority of Pricing Rule, to be applied to an order/item. 
IS_NON_INSTANT_DISCOUNT_RULEChar (1)Indicates if the pricing rule is a Non-Instant Discount pricing Rule 
ENABLE_MIX_N_MATCH_ITEMSChar (1)If set indicates that the rule is applicable for multiple target items. 
COMMON_TRIGGER_QUANTITYNumber (14,4)Indicates quantity required to trigger rule, which all valid trigger items count towards. 
IS_ACTIVE_ON_SUNDAYChar (1)If set indicates that the rule is applicable on Sunday. 
IS_ACTIVE_ON_MONDAYChar (1)If set indicates that the rule is applicable on Monday. 
IS_ACTIVE_ON_TUESDAYChar (1)If set indicates that the rule is applicable on Tuesday. 
IS_ACTIVE_ON_WEDNESDAYChar (1)If set indicates that the rule is applicable on Wednesday. 
IS_ACTIVE_ON_THURSDAYChar (1)If set indicates that the rule is applicable on Thursday. 
IS_ACTIVE_ON_FRIDAYChar (1)If set indicates that the rule is applicable on Friday. 
IS_ACTIVE_ON_SATURDAYChar (1)If set indicates that the rule is applicable on Saturday. 
START_TIMETimeIf not void indicates that the rule is applicable from this time. 
END_TIMETimeIf not void indicates that the rule is applicable till this time. 
IS_FREE_GIFT_RULEChar (1)Indicates whether the rule is a free gift rule or not. 
APPLY_PRORATED_ADJUSTChar (1)When enabled adjustment will be distributed to both the target and the qualifiers for combination rules 
ADJUSTMENT_GRT_UNITPRICEChar (1)if 'N' and the adjustment the rule applies is greater that the target items unitprice, the adjustment will be changed to be the unit price. Default value is 'Y' 
DISCOUNT_ALL_COMBO_ITEMSChar (1)If 'Y' all target items that create a combination will be discounted 
DIST_ADJ_ACROSS_COMBOChar (1)If 'Y' the adjustment for the rule will be distributed across the target combinations 
PRICING_COMPLEXITYVarchar (12)This attribute is used by SBC application. It is a free text field. This attribute is not used by any pricing api internally. 
OVERALL_REDEMPTION_LIMITNumber (14,0)This option limits the number of times that a pricing rule or coupon can be used in total, by all customers 
CUSTOMER_REDEMPTION_LIMITNumber (14,0)This option limits the number of times that a pricing rule or coupon can be used by a single customer. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications. 
CREATETSDateTimeThe timestamp for the creation of this record. 
MODIFYTSDateTimeThe timestamp for the last modification of this record. 
CREATEUSERIDVarchar (40)The user who created this record. 
MODIFYUSERIDVarchar (40)The user who last modified this record. 
CREATEPROGIDVarchar (40)The program that created this record. 
MODIFYPROGIDVarchar (40)The program that modified this record. 
Indexes
