Entity Definition
PLT_CALLOUT
( CONFIGURATION Table )
This table stores metadata for all the callouts.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
CALLOUT_KEYChar (24)The primary key for the PLT_CALLOUT table.  
CALLOUT_NAMEVarchar (255)The name of the CallOut. 
VOTING_RESOLVER_CLASS_NAMEVarchar (128)The java class implementing the voting resolver for the callout.  
CALLOUT_IDVarchar (40)This field is used to uniquely identify a callout. 
PROXY_CLASSVarchar (128)The name of the proxy class. 
IS_INPUT_MODIFIABLEChar (1)This should be set to Y if the input passed to each CallOut unit can be modified; The default value of this field is N which means the input passed to each of the callout units cannot be modified. Any character other than Y will be considered as N 
IS_THREADEDChar (1)This should be set to Y if each CallOut unit needs to be executed in a separate thread; The default value of this field is N which means the callout unit will be executed in the same thread. Any character other than Y will be considered as N 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
