Entity Definition
YFS_EQUIPMENT
( MASTER Table )
All equipments used in the warehouse are defined in this table. The equipment table stores instance of an equipment of a particular type.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
EQUIPMENT_KEYChar (24)The primary key for the YFS_EQUIPMENT table.  
EQUIPMENT_IDVarchar (40)This field identifies the equipment. For example, if the warehouse has 10 fork lifts, the equipment IDs can be FORKLIFT1 to FORKLIFT10. 
EQUIPMENT_TYPEVarchar (40)This field identifies the equipment type, such as FORKLIFT, GRABTRUCK. The equipment types are defined in the YFS_EQUIPMENT_TYPE entity.
A logical foreign key to the following table:
  • YFS_EQUIPMENT_TYPE (in combination with the NODE_KEY column) 
NODE_KEYChar (24)The primary key for the YFS_EQUIPMENT table.
A logical foreign key to the following tables:
  • YFS_SHIP_NODE
  • YFS_EQUIPMENT_TYPE (in combination with the EQUIPMENT_TYPE column) 
CHECK_DIGITVarchar (5)Check Digit is a numeric representation of Equipment. It is generally used by Voice Systems to verify the Equipment. Equipment Check Digit is unique for a Node. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications. 
CREATETSDateTimeThe timestamp for the creation of this record. 
MODIFYTSDateTimeThe timestamp for the last modification of this record. 
CREATEUSERIDVarchar (40)The user who created this record. 
MODIFYUSERIDVarchar (40)The user who last modified this record. 
CREATEPROGIDVarchar (40)The program that created this record. 
MODIFYPROGIDVarchar (40)The program that modified this record.
