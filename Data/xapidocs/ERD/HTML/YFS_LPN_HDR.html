Entity Definition
YFS_LPN_HDR
( TRANSACTION Table )
Stores the LPN and its hierarchy.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
LPN_HDR_KEYChar (24)The primary key for the YFS_LPN_HDR table.  
RECEIPT_HEADER_KEYChar (24)The receipt header key that is associated with this LPN header. This helps identify all stocks attached to a receipt. This number is of significance until the sample from the receipt has been certified as having passed or failed QC. This number is used to pull out all stocks based on the receipt. This is usually flled in situations where the LPN does not have contents.
A logical foreign key to the following table:
  • YFS_RECEIPT_HEADER 
LENGTHNumber (14,4)The length of the LPN. This will be present only if the user has overridden the dimension at LPN creation, typically during receipts. Otherwise, the dimensions are picked up based on the standard packaging type for the item. 
WIDTHNumber (14,4)The width of the LPN. This will be present only if the user has overridden the dimension at LPN creation, typically during receipts. Otherwise, the dimensions are picked up based on the standard packaging type for the item. 
HEIGHTNumber (14,4)The height of the LPN. This will be present only if the user has overridden the dimension at LPN creation, typically during receipts. Otherwise, the dimensions are picked up based on the standard packaging type for the item. 
DIMENSION_UOMVarchar (40)The unit of measure in which the length, width, and height of the LPN are measured. 
WEIGHTNumber (14,4)The weight of the LPN. This will be present only if the user has overridden the weight at LPN creation, typically during receipts. Otherwise, the weight is picked up based on the standard packaging type for the item. 
WEIGHT_UOMVarchar (40)The unit of measure in which the weight of the LPN is measured. 
LPN_NOVarchar (40)A Case Id or Pallet Id. This field is a unique key of YFS_LPN_HDR table. 
PARENT_LPN_NOVarchar (40)The LPN number of the immediate parent of this LPN. 
OUTER_MOST_LPN_NOVarchar (40)The outer most LPN number that is associated with this LPN detail. This is a unique key of YFS_LPN_HDR table.
A logical foreign key to the following table:
  • YFS_LPN_LOCATION 
ORGANIZATION_CODEChar (24)The code of the organization to which the LPN belongs. This is usually filled in situations where the LPN does not have contents. Usually, if we know the items on the LPN, we know the organization to which it belongs. 
CASE_QUANTITYNumber (14,4)The case quantity of the LPN. This will be present only if the user has overridden the case quantity at LPN creation, typically during receipts, because the case quantity is non-standard in this particular LPN. Otherwise, the case quantity is picked up based on the standard packaging type for the item. This attribute is of significance only when we are talking about single SKU pallet LPNs. 
LPN_TYPEVarchar (40)The type of the License Plate Number. For requests involving a license plate, this field is populated as Case or Pallet. The valid values are:
  • CASE: Case
  • PALLET: Pallet 
IS_ALLOCATEDChar (1)This flag indicates whether the complete LPN has been allocated to an operation. 
IS_SHIPMENT_ASSIGNEDChar (1)This field is not currently used. 
SEGMENT_TYPEVarchar (40)The segment type of the inventory. This field indicates the buyer or buyer category for whom the inventory is customized. 
SEGMENTVarchar (40)The segment to which the inventory belongs. This helps to track inventory separately when the inventory is customized for a specific order so that the inventory can be allocated to that order. 
FREEZE_ON_VARIANCEChar (1)This flah indicates whether the LPN had variance detected during a Count operation. This drives whether the LPN can be allocated during Pick Location Assignment. 
TRAN_REQ_NOVarchar (10)A sequence number used to identify the transaction which is in progress for this LPN record. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications. 
IS_OUTBOUND_CONTAINERChar (1)This flag indicates whether the LPN is an Outbound Container.
