Entity Definition
YPM_PRICING_RULE_ASSIGNMENT
( MASTER Table )
This table maintains the assignment of pricing rule to customer and customer attributes.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
PRICING_RULE_ASSIGNMENT_KEYChar (24)The primary key for the YPM_PRICING_RULE_ASSIGNMENT table.  
ASSIGNMENT_IDVarchar (40)Used to UNIQUELY identify a pricing rule assignment. The other columns are still included in the key for backward compatible purposes. Note that existing records, which have NULL for ASSIGNMENT_ID, will have the column populated automatically by SEQ_YPM_ASSIGNMENTID whenever managePricingRuleAssignment on the record. Once populated, it should be solely used. 
PRICING_RULE_KEYChar (24)The pricing rule that this assignment is associated with.
A logical foreign key to the following tables:
  • YPM_COUPON
  • YPM_PRICING_RULE 
ENTERPRISE_CODEChar (24)Organization code of the enterprise to which assignment belongs.
A logical foreign key to the following table:
  • YFS_ORGANIZATION 
CUSTOMER_IDChar (40)The identifier for the customer. If null then this assignment is applicable irrespective of the CustomerID of a customer. 
CUSTOMER_TYPEChar (2)Indicates the type of customer. If null then this assignment is applicable irrespective of the type of a customer. The valid values are:
  • 01: Indicates that this assignment is for business customer.
  • 02: Indicates that this assignment is for consumer customer.
  • 03: Indicates that this assignment is for anonymous customer.
  • 04: Indicates that this assignment is for everyone. 
SHAREABLEChar (1)Indicates whether the rule should be shared with the customer's descendant customers or not. Applicable only when CustomerID is populated on this assignment. 
VERTICALVarchar (40)Vertical of the customer to which the pricing rule is assigned. If null then this assignment is applicable irrespective of the vertical of a customer. 
RELATIONSHIP_TYPEVarchar (40)The type of the relationship that exists between an organization and an enterprise. If null then this assignment is applicable irrespective of the relationship type of a customer. 
CUSTOMER_LEVELVarchar (40)The level of a customer with the enterprise. If null then this assignment is applicable irrespective of the customer level of a customer. 
REGION_KEYChar (24)The region associated to the price list. If null then this assignment is applicable irrespective of the region of a customer.
A logical foreign key to the following table:
  • YFS_REGION 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
