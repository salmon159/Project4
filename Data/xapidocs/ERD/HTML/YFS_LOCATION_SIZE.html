Entity Definition
YFS_LOCATION_SIZE
( MASTER Table )
Each location belongs to a specific location size code. Location size code defines the dimensions of the location, and the weight that it can withstand when products are stored in the location.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

LENGTHNumber (14,4)This field specifies the length of the location. The length is as per the dimension UOM set up against the node's locale.

HEIGHTNumber (14,4)This field indicates the height of location. The height is as per the dimension UOM set up against the node's locale. 
MAX_WEIGHTNumber (14,4)This field indicates the maximum weight that this location can hold. The weight's UOM is as per the weight UOM set up against the node's locale. 
IS_INFINITE_CAPACITYChar (1)This flag indicates whether the dimension related fields are used to compute capacity, and whether the capacity checks are done. 
TRIGGER_PERCENTNumber (5,2)This field is used to trigger inventory monitor if the inventory at the location falls below the percentage of the location capacity that is defined. 
IS_SINGLE_DEEPChar (1)This field indicates whether the location is single deep or multi-deep type.
