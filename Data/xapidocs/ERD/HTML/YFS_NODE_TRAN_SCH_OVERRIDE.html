Entity Definition
YFS_NODE_TRAN_SCH_OVERRIDE
( CONFIGURATION Table )
This table contains information about Node Transfer Schedule Overrides.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
NODE_TRAN_SCH_OVERRIDE_KEYChar (24)The primary key for the YFS_NODE_TRAN_SCH_OVERRIDE table.  
FROM_NODEChar (24)Indicates Node from which transfer originates.
A logical foreign key to the following table:
  • YFS_SHIP_NODE 
TO_NODEChar (24)Indicates Node receiving the transfer.
A logical foreign key to the following table:
  • YFS_SHIP_NODE 
SHIP_DATEDateThe date that has been overridden. This date takes precedence over the transfer schedule. 
CAN_SHIPChar (1)Determines if the transfer can originate on this date. If the value is Y, then it is allowed. Otherwise it is disallowed. Default value: N. 
OVERRIDE_TRANSIT_DAYSNumber (14,4)Transit time for this overridden date. This is only used when CanShip is Y. Default value: 0. 
TRANSIT_UOMVarchar (40)This determines the UOM used for this transfer schedule override. If null, then it is assumed to be DAY. Possible values: DAY, HR 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.

MODIFYTSDateTimeThe timestamp for the last modification of this record. 
CUT_OFF_TIMETimeCut off time for Node transfer schedule for the overridden date

CREATEPROGIDVarchar (40)The program that created this record.
