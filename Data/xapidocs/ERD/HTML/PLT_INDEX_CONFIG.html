Entity Definition
PLT_INDEX_CONFIG
( CONFIGURATION Table )
This table contains status of index



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
INDEX_CONFIG_KEYChar (24)The primary key for the PLT_INDEX_CONFIG table.  
ENTERPRISE_CODEChar (24)Organization code of the enterprise to which the index belongs. 
IS_CUSTOM_MODEChar (1)This flag indicates whether the index follows the definition in a configuration xml. Valid Values : null (if index is defined in the corresponding entity definition xml) or Y (if index is defined in a configuration xml). 
INDEX_NAMEVarchar (25)The actual name of the index if index is defined in the entity xml (non custom mode) or the index config identifer defined in a configuration xml (custom mode). 
INDEX_VERSIONNumber (10,0)This field indicates the version of index definition.  
INDEX_TEMPLATECLOBThe template for the index. This is used to retrieve the data to index. In case of custom mode it holds the index definition identified the indexConfigId and for non-custom mode this contains an entity output template with all indexable attributes. 
INDEX_UPDATE_REQUIREDChar (1)This flag indicates whether the recent update is done on the record 
CREATEUSERIDVarchar (40)User who created the record. 
MODIFYUSERIDVarchar (40)User who modified the record. 
CREATEPROGIDVarchar (40)Program that created the record. 
MODIFYPROGIDVarchar (40)Program that modified the record. 
CREATETSDateTimeTimestamp when the record was created. 
MODIFYTSDateTimeTimestamp when the record was last modified. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
