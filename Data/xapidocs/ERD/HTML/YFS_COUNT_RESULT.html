Entity Definition
YFS_COUNT_RESULT
( TRANSACTION Table )
This transaction table stores results of the count requests.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
COUNT_RESULT_KEYChar (24)The primary key for the YFS_COUNT_RESULT table.  
COUNT_REQUEST_KEYChar (24)The count request key that is associated with this count result.
A logical foreign key to the following table:
  • YFS_COUNT_REQUEST 
TASK_IDVarchar (40)The task id that is associated with this count result. 
SUMMARY_TASK_IDVarchar (40)The summary task identifier associated with the count result. 
TASK_TYPEChar (10)This field is a two-digit code that identifies the task type. 
NODE_KEYChar (24)The node key that is associated with this count result. 
ORGANIZATION_CODEChar (24)The organization to which the inventory belongs. 
LOCATION_IDVarchar (40)A Location in the node where count was performed. 
ORIGINAL_LOCATIONVarchar (40)The original location of inventory counted. 
LPN_NOVarchar (40)The case/pallet License Plate Number for which count results are recorded. 
PARENT_LPN_NOVarchar (40)The LPN that has includes other LPN inside it. 
LPN_TYPEVarchar (40)The system defined values for the LPN type associated with the LPN number for which the count results are recorded. Currently, the system supports only CASE and PALLET LPN types. The valid values are:
  • CASE: Case
  • PALLET: Pallet 
ITEM_IDChar (40)This field indicates the item that was counted. 
UOMVarchar (40)This field indicates the item's unit of measure. 
PRODUCT_CLASSChar (10)The inventory classification of the item that is classified based on the product characteristics. 
SERIAL_NOVarchar (40)This field indicates the serial number that was generated during the count process. 
INVENTORY_STATUSChar (10)This field is a unique key for the YFS_INVENTORY_STATUS table. This field refers to an inventory sub-classification based on the inventory control process results within the warehouse. 
SEGMENT_TYPEVarchar (40)Whenever inventory is customized, it needs to be tracked separately. This field indicates the buyer or buyer category for whom the inventory is customized. 
SEGMENTVarchar (40)Whenever the inventory is customized for a specific order, the inventory needs to be tracked separately. This field indicates the order to which the inventory is allocated. 
SHIP_BY_DATEDateThe date by which the item must be shipped so that it is usable. This is also referred to as the expiration date. 
INVENTORY_TAG_KEYChar (24)The inventory tag key that is associated with this count result. 
TAG_NUMBERVarchar (120)This field indicates the tag number associated with the inventory. 
SYSTEM_QUANTITYNumber (14,4)This field indicates the system reported quantity for the count criteria. 
COUNT_QUANTITYNumber (14,4)This field indicates the physically counted quantity for the count criteria. 
VARIANCE_QUANTITYNumber (14,4)This field specifies the difference between the count quantity and system quantity. 
VARIANCE_PERCENTAGENumber (15,2)This field specifies the variance percentage which is calculated as 100* VARIANCE_QUANTITY divided by SYSTEM_QUANTITY. 
VARIANCE_TYPEVarchar (20)This field determines the variance type based on the count results. If the Variance Quantity > 0, the Variance Type for the items will be 'New', 'Missing' if the Variance Quantity < 0, and 'No Variance' if the Variance Quantity = 0. However, for License Plates the Variance Type will not be computed based on the Variance Quantity. Instead, the Variance Type will be set to 'New' if a Case/Pallet is found in the location, but does not exist in the system inventory of the location, 'Missing' if the Case/Pallet is not found in the location, but exists in the system inventory of the location, 'No Variance' in situations when there is no count discrepancy. The valid values are:
  • MISSING: Missing
  • NEW: New
  • NO VARIANCE: No Variance 
VARIANCE_VALUENumber (19,2)This field is computed as the summation of VARIANCE_QUANTITY * UNIT_COST for all tasks in the count request for the latest completed iteration. 
VARIANCE_ACCEPTEDChar (1)This field indicates if the variance was accepted for the count result. 
IS_RESOLVABLEChar (1)This flag indicates whether the variance can be resolved for the count result. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.

MODIFYTSDateTimeThe timestamp for the last modification of this record. 
CREATEUSERIDVarchar (40)The user who created this record. 
MODIFYUSERIDVarchar (40)The user who last modified this record. 
CREATEPROGIDVarchar (40)The program that created this record. 
MODIFYPROGIDVarchar (40)The program that modified this record.
