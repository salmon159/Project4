Entity Definition
PLT_PROPERTY
( CONFIGURATION Table )
The values for each property is stored here.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
PROPERTY_KEYChar (24)The primary key for the PLT_PROPERTY table.  
BASE_PROPERTY_NAMEVarchar (255)The name of the base property.
A logical foreign key to the following table:
  • PLT_PROPERTY_METADATA (in combination with the CATEGORY column) 
CATEGORYVarchar (20)Category to which the property belongs.
A logical foreign key to the following table:
  • PLT_PROPERTY_METADATA (in combination with the BASE_PROPERTY_NAME column) 
PROPERTY_OVERRIDEVarchar (100)Specifies who has overridden the property. If this is 'BASE', it means that this is the base property. Allowed values are SERVER, BASE and USER. 
PROPERTY_OVERRIDE_NAMEVarchar (100)The overriden name of the property. For base property this would be blank. 
PROPERTY_VALUEVarchar (1000)The value of the property. 
FACTORY_VALUEVarchar (1000)The value of the property shipped out. 
USER_COMMENTVarchar (200)To store comments when modifying or overriding a property. 
CREATETSDateTimeThe timestamp for the creation of this record. 
MODIFYTSDateTimeThe timestamp for the last modification of this record. 
CREATEUSERIDVarchar (40)The user who created this record. 
MODIFYUSERIDVarchar (40)The user who last modified this record. 
CREATEPROGIDVarchar (40)The program that created this record. 
MODIFYPROGIDVarchar (40)The program that modified this record. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
