Entity Definition
YFS_CHANGE_REQUEST
( CONFIGURATION Table )
This table stores Change Request definitions.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
CHANGE_REQUEST_KEYChar (24)The primary key for the YFS_CHANGE_REQUEST table.  
CHANGE_PROJECT_KEYChar (24)The reference key to the YFS_CHANGE_PROJECT table.
A logical foreign key to the following table:
  • YFS_CHANGE_PROJECT 
CHANGE_REQUEST_IDVarchar (24)The identifier for this change request. If it is not passed during creation, it is generated. It cannot be set to DEFAULT. 
MANAGE_USER_IDVarchar (50)Identifier for the user who caused a change in the Change Project. This is always taken from the Environment. 
STATUSVarchar (10)The status of the Change Request. As the request progresses through the change process, the status changes. The valid values are:
  • 1000: Draft
  • 1100: Open
  • 1100.50: Work In Progress
  • 1200: Completed
  • 1300: Published 
SHORT_DESCRIPTIONVarchar (100)The short description of the Change Request. 
LONG_DESCRIPTIONVarchar (255)The long description of the Change Request. 
CREATETSDateTimeThe timestamp for the creation of this record. 
MODIFYTSDateTimeThe timestamp for the last modification of this record. 
CREATEUSERIDVarchar (40)The user who created this record. 
MODIFYUSERIDVarchar (40)The user who last modified this record. 
CREATEPROGIDVarchar (40)The program that created this record. 
MODIFYPROGIDVarchar (40)The program that modified this record. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
