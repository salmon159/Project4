Entity Definition
YFS_CUSTOMER_SUBSCRIPTION
( MASTER Table )
This table stores all customer subscriptions in the system.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription

CUSTOMER_IDChar (40)The identifier for the customer.
A logical foreign key to the following table:

A logical foreign key to the following table:

SUBSCRIPTION_NOVarchar (40)The number of the subscription. 
CUSTOMER_EMAILIDVarchar (150)The E-mail ID of the customer. 
CUSTOMER_CONTACT_IDVarchar (40)This is the customer contact ID for the subscription. This field maps to contact ID field of YFS_Customer_Contact. 
SELLER_ORGANIZATION_CODEChar (24)The seller organization that is associated with this subscription. This must be a valid organization defined in the system. 
SUBSCRIPTION_NAMEVarchar (100)The name given to the subscription.  
SUBSCRIPTION_DATEDateTimeThe date that the subscription was created. 
START_DATEDateTimeThe date on which the subscription period started. 
END_DATEDateTimeThe end date of the subscription. 
ITEM_IDChar (40)The identifier for the item. 
ITEM_DESCRIPTIONVarchar (500)The description of the item. 
ITEM_SHORT_DESCRIPTIONVarchar (200)The short description of the item. 
ITEM_GROUP_CODEVarchar (20)This field indicates what kind of item is on the order line. The valid values are:
  • PROD: Indicates that the item is a product item.
  • PS: Indicates that the item is a provided service item.
  • DS: Indicates that the item is a delivery service item.
  • SS: Indicates that the item is a special service item. 
UOMVarchar (40)The unit of measure. 
QUANTITYNumber (14,4)The quantity of the line item that has been subscribed. 
STATUSVarchar (4)The status of the subscription. The valid values are:
  • 1000: Pending Activation
  • 1100: Active
  • 1200: Suspended
  • 9000: Disconnected 
SUBSCRIPTION_TYPEVarchar (20)The type of the subscription. 
ASSOCIATED_SUBSCRIPTION_KEYChar (24)Subscription key of the new subscription created from this subscription. 
KIT_CODEVarchar (20)The collection of items that form a unit. For example, a home theatre system is a unit consisting of kit components, such as a DVD player, TV, CD player, etc. The valid values are:
  • LK: Indicates a logical kit.
  • PK: Indicates a physical kit.
  • DPK: Indicates a dynamic physical kit.
  • BUNDLE: Indicates a bundle item. 
BUNDLE_PARENT_SUBSCRIPTION_KEYChar (24)The Bundle parent's subscription key. If this line is a bundle component, this key will point to the parent subscription. 
KIT_QTYNumber (14,4)The quantity that is used to compute ordered quantity for bundle component line. It can be specified only for bundle components. If specified ordered quantity will be set to KitQty*OrderedQty of the bundle parent. 
BOM_CONFIG_KEYChar (24)This field is used by Sterling Multi-Channel Selling Solution so that Visual Modeler can be opened with existing options. 
UNIT_PRICENumber (19,6)The unit price for the order line. 
LIST_PRICENumber (19,6)the list price of the item. This is an informational field. 
TOTAL_AMOUNTNumber (15,2)The grand total for the subscription. 
CURRENCYVarchar (20)The currency code in which this subscription is processed. All price fields are assumed to be in this currency.
