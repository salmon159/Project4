Entity Definition
YFS_LOCATION
( MASTER Table )
Warehouse zones are further divided into locations of different sizes where inventory is stored. Each location belongs to a zone and a node. It is classified as INTRANSIT, REGULAR, VIRTUAL, STAGING or DOCK. It has attributes like aisle number, bay number, level number etc.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

LOCATION_BAR_CODEVarchar (40)This field indicates that a numeric barcode of the location identifier is stored for this location. The way this is intended to be used in WMS is that an RF operator when prompted to scan the location Id is also expected to scan the location bar code on the RF terminal. This ensures that the operator has traveled to the location suggested by the system and picked up or dropped off inventory at the specified location. Simply keying in the human readable form of the location ID on the RF terminal will be rejected by the system. The objective of this feature is to ensure inventory accuracy. 
LOCATION_TYPEVarchar (40)This field is a system defined classification of location to aid association of locations of certain types for certain other operations with WMS. The supported types are: INTRANSIT (Mobile locations), STAGING, VIRTUAL, REGULAR and DOCK. For example, all equipment locations should be of type INTRANSIT. The valid values are:
  • INTRANSIT: In Transit Location
  • STAGING: Staging Location
  • VIRTUAL: Virtual Location
  • REGULAR: Regular Location
  • DOCK: Dock Location 
ZONE_IDVarchar (40)This field indicates the zone to which a location belongs. This when associated with the node key identifies a unique zone within the node.
A logical foreign key to the following table:
  • YFS_ZONE (in combination with the NODE_KEY column) 
AISLE_NUMBERNumber (9,0)This field indicates the aisle Number of the location. The locations belong to zones which have travel aisle's between them. A zone could belong to multiple aisles and multiple zones could belong to an aisle. But a location in a zone always belongs to one and only one aisle. 
AISLE_SUFFIXVarchar (40)this field indicates the aisle section of the location id. 
LEVEL_NUMBERNumber (9,0)This field indicates the level number of the location. The height of the location (y-co-ordinate of the location from the floor) is classified as levels. The level attribute of the location is used in arriving at locations nearest to the dedicated locations algorithm used during the putaway process. Typically, the level attribute is contained within the location ID. 
BAY_NUMBERNumber (9,0)This field indicates the location's bay number. Typically, the aisle, level and bay put together represents the physical location of the location in a node, if they are based on coordinate system. The bay attribute of the location (x-coordinate from the beginning of the aisle) is used in arriving at locations nearest to the dedicated locations algorithm that is used in putaway. Typically, the bay attribute is contained within the location ID. 
FREEZE_MOVE_INChar (1)This flag indicates whether the inventory is moved into the location. This is used for warehouses with certain number of locations. However, the usage of locations is phased out over a period of time. 
MOVE_IN_SEQ_NONumber (9,0)This field is used by task management to suggest location while moving in the inventory (putaway). The putaway location selection algorithm uses this information to select locations from a list of locations based on the move in sequence. 
FREEZE_MOVE_OUTChar (1)This flag indicates whether inventory is moved out of the location. 
MOVE_OUT_SEQ_NONumber (9,0)This field is used by task management to suggest location while moving out inventory (picking). The pick location selection algorithm uses this information to select locations from a list of locations based on its move in sequence. 
IN_STAGING_LOCATION_IDVarchar (40)This field indicates that the inventory is dropped into the drop off locations. 
OUT_STAGING_LOCATION_IDVarchar (40)This field indicates the Out Drop off Location where moves originated at this location, and may be dropped. 
VELOCITY_CODEVarchar (40)This field indicates that the items are classified as 'A', 'B' or 'C' class items based on whether they are fast selling, not so fast selling and low selling items. These item classifications are typically followed by all enterprises to optimize certain operations such as sourcing and stocking. The reason to have locations that prefer certain velocity codes is to have locations closer to dock stocking 'A' class items, and locations furthest away from the dock stocking 'C' class items. The velocity code is a preference on the location and not a constraint. If 'A' class items fill up all locations meant for 'A' class items, then they can be moved to 'B' and later 'C'. Similarly, 'C' can go to 'B' and 'A' for lack of space in the respective locations preferred for a specific velocity code. The 'B' class items move into 'C' and then into 'A'. 
LOCATION_SIZE_CODEVarchar (40)This field defines the capacity of the location. All locations having the same size (dimensions and ability to hold the same weight) are classified under the same size code. This maps to the primary key attribute of the YFS_LOCATION_SIZE table.
A logical foreign key to the following table:
  • YFS_LOCATION_SIZE (in combination with the NODE_KEY column) 
STORAGE_CODEVarchar (40)This field is an attribute of the location that allows the warehouse to store items with similar storage profile as that of the location. For example, hazardous inflammable items need locations close to fire extinguishers. In this case the locations will be marked as having a storage code, which is suitable for storing Inflammable items. This ensures that only inflammable items get to these locations.
A logical foreign key to the following table:
  • YFS_LOCATION_STORAGE_CODE (in combination with the NODE_KEY column) 
DOCK_TYPEVarchar (10)This field depicts the type of the dock location. For a non-dock location, this field is set to blank. For a dock location, it is set to INBOUND if it is used exclusively for inbound activities. Similarly, it is set to OUTBOUND if is used exclusively for outbound activities. In case, the dock location is used for both inbound and outbound activities, this attribute is set to BOTH. 
X_CO_ORDINATENumber (14,4)This field indicates the X co-ordinate of the location. 
Y_CO_ORDINATENumber (14,4)This field indicates the Y co-ordinate of the location. 
Z_CO_ORDINATENumber (14,4)This field indicates the Z co-ordinate of the location. 
CHECK_DIGITVarchar (5)Check Digit is a numeric representation of Location. It is generally used by Voice Systems to verify the Location. Location Check Digit is non unique. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
