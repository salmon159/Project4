Entity Definition
YFS_ORDER_VALIDATION
( CONFIGURATION Table )
This table stores defaulting rules and validations for seller organization, buyer organization, customer identification, vendor identification, and item during document creation for a particular Enterprise and document type combination.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription

ORGANIZATION_CODEChar (24)Organization code for which the rules will be applied to.
A logical foreign key to the following table:

DOCUMENT_TYPEVarchar (40)Type of document the rules will be applied to. 
SELLER_VALIDATIONChar (2)Type of validation on what can be the value for the seller on the document. This field can take one of the pre-defined values (00 - No validation, 01 - Same as enterprise, 02 - Defined in enterprise's hierarchy, 03 - Customer or Vendor of the enterprise). 
BUYER_VALIDATIONChar (2)Type of validation on what can be the value for the buyer on the document. This field can take one of the pre-defined values (00 - No validation, 01 - Same as enterprise, 02 - Defined in enterprise's hierarchy, 03 - Customer or Vendor of the enterprise). 
VALIDATE_CUSTOMER_IDChar (1)Flag determines whether to validate CustomerID or not during document creation. 
VALIDATE_VENDOR_IDChar (1)Flag determines whether to validate VendorID or not during document creation. 
VALIDATE_ITEMChar (1)Flag determines whether to validate item or not during document creation.
