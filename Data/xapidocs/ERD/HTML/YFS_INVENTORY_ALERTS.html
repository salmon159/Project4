Entity Definition
YFS_INVENTORY_ALERTS
( TRANSACTION Table )
Stores information on inventory alerts raised by the system.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
INVENTORY_ALERT_KEYChar (24)The primary key for the YFS_INVENTORY_ALERTS table.  
INVENTORY_ITEM_KEYChar (24)Inventory item for which alert was raised for. 
SHIP_NODEChar (24)Ship node. 
ALERT_TYPEVarchar (40)Type of alert. The valid values are:
  • ONHAND: Deprecated. ATP_MONITOR for on hand availability.
  • CURRENT: Deprecated. ATP_MONITOR for current availability.
  • FUTURE: Deprecated. ATP_MONITOR for future availability.
  • REALTIME_ONHAND: REALTIME_ATP_MONITOR for on hand availability.
  • REALTIME_FUTURE_MAX: REALTIME_ATP_MONITOR for future maximum inventory within the horizon. 
ALERT_RAISED_ONDateTimeDate alert was raised. 
ALERT_PERIOD_STARTDateDeprecated. Start date of period for which alert was raised using the INVENTORY_MONITOR. 
ALERT_PERIOD_ENDDateDeprecated. End date of period for which alert was raised using the INVENTORY_MONITOR. 
ALERT_QUANTITYNumber (14,4)Available Quantity corresponding to the AlertLevel. 
ALERT_LEVELVarchar (20)Level of alert raised. 3 for low, 2 for medium, 1 for high, and 0 for thresholds greater. 
ALERT_ACTIONChar (24)Deprecated. Action raised using the INVENTORY_MONITOR. 
ONHAND_AVAILABLE_QUANTITYNumber (14,4)Total on-hand inventory available quantity. If inventory is not tracked, unplanned, or infinite, the value may be Java's Integer.MAX_VALUE, Java's Long.MAX_VALUE, or the RealisticQuantityForInfiniteInventory property (if set). 
ONHAND_AVAILABLE_DATEDateDate when on-hand inventory quantity is available. 
FUTURE_AVAILABLE_QUANTITYNumber (14,4)Maximum future inventory available quantity within the monitoring period. The end of the monitoring period is determined by the lead time override (if specifies) in the monitor rule or the lead time of the monitoring item. If inventory is not tracked, unplanned, or infinite, the value may be Java's Integer.MAX_VALUE, Java's Long.MAX_VALUE, or the RealisticQuantityForInfiniteInventory property (if set). 
FIRST_FUTURE_AVAILABLE_DATEDateEarliest date when any future inventory will be available. 
FUTURE_AVAILABLE_DATEDateDate when the maximum future inventory will be available. 
DISTRIBUTION_RULE_IDChar (40)The unique identifier for a distribution rule. 
IS_DEFAULT_DISTRIBUTION_GROUPChar (1)Deprecated. This flag indicates if the distribution rule id is the default distribution rule id for the inventory organization running the REALTIME_ATP_MONITOR. 
AVAILABILITY_INFOCLOBThis field contains availability information matrix containing ship dates and corresponding available quantity for each date. 
ORGANIZATION_CODEChar (24)The distribution rule ID owner the alert was raised for. 
LATITUDENumber (11,8)The latitude of the node for which this alert. If the alert is for a distribution group, then this will be null. 
LONGITUDENumber (11,8)The longitude of the node for which this alert. If the alert is for a distribution group, then this will be null. 
ESTIMATED_COSTNumber (19,6)The estimated cost of the item to be fulfilled by the ShipNode. Includes item cost and outbound handling cost. 
DELIVERY_METHODVarchar (40)The delivery method used for computing the alert information. 
UNASSIGNED_ONHAND_QUANTITYNumber (14,4)The amount of unassgined on hand quantity that is being balanced to the distribution group. Internal purposes only. 
UNASSIGNED_FUTURE_QUANTITYNumber (14,4)The amount of unassgined future quantity that is being balanced to the distribution group. Internal purposes only. 
PROCURED_QTYNumber (14,4)The amount of quantity procured to accommodate inventory available. 
PROCUREMENT_LEAD_TIMENumber (14,4)The amount of time needed for procurement. This is the transit time and receipt processing times added together. This is measured in hours. 
CREATETSDateTimeTimestamp when the record was created. 
MODIFYTSDateTimeTimestamp when the record was last modified. 
CREATEUSERIDVarchar (40)User who created the record. 
MODIFYUSERIDVarchar (40)User who modified the record. 
CREATEPROGIDVarchar (40)Program that created the record. 
MODIFYPROGIDVarchar (40)Program that modified the record. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
