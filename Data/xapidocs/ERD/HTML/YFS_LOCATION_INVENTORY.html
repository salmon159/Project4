Entity Definition
YFS_LOCATION_INVENTORY
( TRANSACTION Table )
Non-license plated inventory is tracked using this table.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
LOCATION_INVENTORY_KEYChar (24)The primary key for the YFS_LOCATION_INVENTORY table.  
NODE_KEYChar (24)The node key that is associated with this location inventory.
A logical foreign key to the following table:
  • YFS_LOCATION (in combination with the LOCATION_ID column) 
ZONE_IDVarchar (40)The zone id that is associated with this location. This is the unique key attribute of the YFS_ZONE table in conjunction with NODE_KEY. 
LOCATION_IDVarchar (40)The identifier for the location. This, in conjunction with NODE_KEY, identifies a unique location in the node.
A logical foreign key to the following table:
  • YFS_LOCATION (in combination with the NODE_KEY column) 
INVENTORY_ITEM_KEYChar (24)The inventory item key that is associated with this location inventory. This points to a SKU that comprises the Inventory Organization, Item ID, Product Class and Unit of Measure.
A logical foreign key to the following table:
  • YFS_INVENTORY_ITEM 
INVENTORY_TAG_KEYChar (24)The inventory tag key that is associated with this location inventory. This points to lot and related attributes (revision etc) for the item whose inventory is tracked here.
A logical foreign key to the following table:
  • YFS_INVENTORY_TAG 
INVENTORY_STATUSChar (10)The inventory status that is associated with this location inventory. This refers to an inventory sub classification based on the results of the inventory control processes within the warehouse. 
SEGMENT_TYPEVarchar (40)The segment type of the inventory. This field indicates the buyer or buyer category for whom the inventory is customized. 
SEGMENTVarchar (40)The segment to which the inventory belongs. This helps to track inventory separately when the inventory is customized for a specific order so that the inventory can be allocated to that order. 
FIFO_NONumber (9,0)A date based inventory attribute that helps understand the order in which stock arrived at the node. This is used to send out items that arrived earlier than the ones that arrived later. 
RECEIPT_HEADER_KEYChar (24)The receipt header key that is associated with this LPN header. This helps identify all stocks attached to a receipt. This number is of significance until the sample from the receipt has been certified as having passed or failed QC. This number is used to pull out all stocks based on the receipt. This is usually flled in situations where the LPN does not have contents. 
COUNTRY_OF_ORIGINVarchar (40)The country in which the item was manufactured. 
SHIP_BY_DATEDateThe date by which the item must be shipped so that it is usable. This is also referred to as the expiration date. 
QUANTITYNumber (14,4)The number of units of the item available at this location. 
PEND_IN_QTYNumber (14,4)The number of units of the item waiting to move into this location. 
HARD_ALLOC_QTYNumber (14,4)The number of units of this item waiting to move out of this location. This is typically done to service an order. This item will move out of this location for the specified purpose. 
SOFT_ALLOC_QTYNumber (14,4)The number of units of this item waiting to move out of this location. However, it is not necessary that it may move because soft allocations are typically suggestions made by the system for space optimization purposes or for operations within the warehouse not related to order shipments. 
SHIPMENT_ASSIGNED_QTYNumber (14,4)This field is not currently used. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications. 
CREATETSDateTimeThe timestamp for the creation of this record. 
MODIFYTSDateTimeThe timestamp for the last modification of this record. 
CREATEUSERIDVarchar (40)The user who created this record. 
MODIFYUSERIDVarchar (40)The user who last modified this record. 
CREATEPROGIDVarchar (40)The program that created this record. 
MODIFYPROGIDVarchar (40)The program that modified this record.
