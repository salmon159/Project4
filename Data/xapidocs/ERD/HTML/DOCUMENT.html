Entity Definition
DOCUMENT
( TRANSACTION Table )
The Document represents incoming data (e.g.,HTML or XML document) to the Workflow system. Conceptually, it is the handle of the persisted document in permanent storage (i.e., the database), and provides the primary access to the document's data. \n Note: Once a Document object has been persisted, it is viewed as Read Only. Changes to a persisted Document object will result in creation of a new document and will get persisted with a distinct document Id.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.
