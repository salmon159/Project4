Entity Definition
YCS_COMMODITY_RECORD
( TRANSACTION Table )
This table stores item level information for an international UPS shipper. So there will be one record for each item of the carton in the manifest.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
COMMODITY_RECORD_KEYChar (24)The primary key for the YCS_COMMODITY_RECORD table.  
MANIFEST_NUMBERChar (30)This has the MANIFEST_UPS_DTL_KEY of the carton (the carton to which the item belongs) in the YCS_MANIFEST_UPS_DTL table. 
INVOICE_LINE_NUMBERChar (3)The line number associated with the commodity or invoice line as indicated on the commercial invoice. 
COMMODITY_CODEChar (30)The harmonized code representing the commodity as indicated on the commercail invoice. 
PART_NUMBERChar (35)The shipper's part number or reference number for the commodity as indicated on the commercial invoice. 
LINE_ORIGIN_COUNTRYChar (2)The country of manufacture or production or growth of the article. Must coform to ISO/IATA country code. 
LINE_CURRENCY_CODEChar (3)The Currency Code for the value of the commodity contained in the invoice line as indicated on the commercial invoice. 
ECCNChar (15)The Export Commodity Control Number for the commodity. 
LINE_UNIT_AMT_PRICENumber (22,4)The per unit value of the commodity in the invoice line as indicated on the commercial invoice. 
LINE_QUANTITYNumber (22,4)The quantity of the commodity in the invoice line as indicated on the commercial invoice. 
LINE_QTY_UOMChar (3)The unit of measure of the commodity in the invoice line as indicated on the commercial invoice. 
LINE_LICENSE_INFOChar (35)The validated export license number for the commodity contained in the shipment, as indicated on the Waybill and/or S.E.D. . This field is currently used for US and PR origins only, as specified by UPS. Required if the SED_Code equals D or U and shipment origin is US or PR. 
LINE_LICENSE_EXP_DATEDateThe expiry date for the validated export license, as indicated on the Waybill and/or S.E.D. If LINE_LICENSE_INFO is an exception, this is set to spaces. 
LINE_MERCH_DESC1Char (35)The detail description of the commodity in the invoice line, as indicated on the commercial/customs invoice. 
LINE_MERCH_DESC2Char (35)Continuation of the detail description of the commodity in the invoice line, as indicated on the commercial/customs invoice. 
LINE_MERCH_DESC3Char (35)Continuation of the detail description of the commodity in the invoice line, as indicated on the commercial/customs invoice. 
CERT_OF_ORIGIN_NOChar (10)The number found on Certificate Of Origin. 
CERT_OF_ORIGIN_CODEChar (5)The code representing the type of origin certification that applies to the commodity. 
AGREEMENT_TYPEVarchar (10)The types of trade agreements that are applicable for the commodities contained in the invoice line, as indicated on the commercial/customs invoice. EEC European Economic Community EFTA European Free Trade Agreement NAFTA North American Free Trade Agreement. 
COMMODITY_REMARKSVarchar (200)Commodity level remarks produced by the customer. Example: Commodity Level Declaration Statements. 
QUANTITY_SCHEDULE_BUNITSNumber (22,4)The quantity of the commodity contained in the SED (Shipper's Export Declaration) line, as indicated in the SED document. This field is used for US and PR origins only, as specified by UPS. 
MARKS_NUMBERSChar (35)Any distinguishing marks or numbers that appear on the package. This field is valid only for US, PR, CA and VI shippers, as specified by UPS. 
COMMODITY_WEIGHTNumber (22,4)The gross commodity weight for the SED and/or CO document. This field must be greater than zero if SED_CODE or COCODE in YCS_INTERNATIONAL_RECORD is not blank. 
NUMBER_OF_PACKAGES_PER_CMMDTYNumber (22,4)The number of packages in the shipment containing this commodity. This field is currently used for US and PR origins, as specified by UPS. 
SED_LINE_AMTNumber (22,4)The value of the SED Commodity contained on the SED. As per UPS specification, this field is currently used for US and PR origins only and this field must be greater than zero if SED_CODE in YCS_INTERNATIONAL_RECORD is not blank. 
CO_TYPENumber (22,4)The Certificate Of Origin Type. As per UPS specification, this field is currently used for US and PR origins only; if COCODE in YCS_INTERNATIONAL_RECORD is not blank, valid values are 1 ( Certificate of Origin), 2 (NAFTA Certificate of Origin); if COCODE is blank, this is set to space. 
SED_INDNumber (22,4)This indicates if SED is applicable.As per UPS specification, this field is currently used for US and PR origins only; if SED_CODE in YCS_INTERNATIONAL_RECORD is not blank, valid value is 1; else 0. 
LINE_EXTENDED_AMTNumber (22,4)The value of the commodity in the invoice line, as indicated on the commercial invoice. This equals the LINE_UNIT_AMT_PRICE multiplied by the LINE_QUANTITY. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
