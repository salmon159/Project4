Entity Definition
YFS_IBA_TRIGGER
( TRANSACTION Table )
This table stores information for inventory item and node, which were triggered to be processed by the Item Based Allocation agent.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

A logical foreign key to the following table:

IBA_REQUIREDChar (1)This flag indicates that Item Based Allocation process is required for this item and node combination. Valid Values: Y/N. 
IBA_RUN_REQUIREDChar (1)This flag indicates that Item Based Allocation agent must process this trigger. This will not be set to Y unless the IBA_REQUIRED field is set to Y. Valid Values: Y/N. 
PROCESSING_BY_AGENTChar (1)This flag indicates that this trigger is being processed by the Item Based Allocation agent. 
PROCESS_OVER_BY_TSDateTimeThe timestamp that system expects the Item Based Allocation process to be over by. System will treat the PROCESSING_BY_AGENT flag as N regardless of the actual value when current timestamp is after this timestamp. This can be used in the scenario when the Item Based Allocation agent does not complete its operation due to some failures and the PROCESSING_BY_AGENT flag does not get reset to N. Default Value: current timestamp. 
LAST_IBA_PROCESSED_TSDateTimeThe timestamp of the last time that the Item Based Allocation agent successfully processed this trigger.
