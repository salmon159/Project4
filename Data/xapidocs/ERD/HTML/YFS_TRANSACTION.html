Entity Definition
YFS_TRANSACTION
( CONFIGURATION Table )
This table stores transactions defined within the system.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

IS_MANUALChar (1)This flag indicates whether this is a manual transaction requiring human intervention. 
LISTENER_TYPEChar (2)This field indicates the type of the listener transaction. Each process type to process type combination consistutes a different listener type. The valid values are:
  • 01: Indicates the chained order to sales order listener type.
  • 02: Indicates the derived order to sales order listener type.
  • 03: Indicates the shipment to order listener type.
  • 04: Indicates the load to shipment listener type.
  • 05: Indicates the receipt to shipment listener type.
  • 06: Indicates the receipt to order listener type.
  • 07: Indicates the work order to container listener type.
  • 08: Indicates the work order to order listener type.

  • N: Indicates that it is not required.
  • S: Indicates a single instance of each DERIVED_DOCUMENT_TYPE for a process type and base transaction.
  • M: Indicates multiple instances of each DERIVED_DOCUMENT_TYPE for a process type and base transaction. 
DERIVED_DOCUMENT_TYPEChar (24)The derived document type.

  • ALL: Indicates that derived transactions can have any pickup statuses.
  • EXACT: Indicates that derived transactions can only have base pickup statuses.
  • BASE: Indicates that derived transactions can only have base pickup statuses. 
DROP_STATUS_FILTERChar (5)This field indicates how drop statuses will be filtered. The valid values are:
  • ALL: Indicates that derived transactions can have any drop statuses.
  • EXACT: Indiates that derived transactions can only have base drop statuses.
  • EXTN: Indicates that derived transactions can have base drop statuses or any of its extensions. 
SINGLE_DROP_STATUSChar (1)This flag indicates whether this transaction can only have one drop status. 
EXTERNALLY_TRIGGERABLEChar (1)This flag indicates whether this transaction is externally triggerable. 
TIME_TRIGGERABLEChar (1)This flag indicates whether this transaction is time triggerable. 
USER_TRIGGERABLEChar (1)This flag indicates whether this transaction is user-triggerable. 
EXTERNALLY_TRIGGEREDChar (1)This flag indicates whether the transaction is externally triggered. 
TIME_TRIGGEREDChar (1)This flag indicates whether the transaction is time triggered. 
AGENT_JAVA_CLASSVarchar (128)The java class used for consuming work flow messages being sent to a pipeline.

BASE_PROCESS_TYPE_LISTENED_TOChar (24)The base process type that the transaction listens to for status changes that may be needed to apply status changes to other entities.  
HOLD_TYPE_ENABLEDChar (1)This flag indicates whether this transaction is capable of understanding hold types. Default value is 'N'. 
SUPPORTS_DEPENDENCYChar (1)This flag indicates whether transaction dependencies can be specified for this transaction. 
SUPPORTS_TRAN_COMPLETIONChar (1)This flag indicates whether transaction completion can be recorded for this transaction

CAN_RESOLVE_HOLD_TYPEChar (1)This flag indicates whether this transaction is capable of resolving hold types. Default value is 'N'. 
HOLD_PREVENTION_LEVELVarchar (40)The level at which a transaction can be prevented by a hold. Possible values are 'ORDER' and 'ORDER_LINE'. 
SUPPORTS_OPERATION_LEVELChar (1)This is a boolean that identifies whether a transaction supports operation level. For a non-derived user transaction, this is N. For a derived user transaction, the value is carried over from the abstract base transaction . 
OPERATION_LEVELVarchar (20) This is a common code and application defined. This column indicates the level at which the transaction operates. In specific applications, the operation level can be assigned at either the header or at line level of the transaction. When the header level operation is used, all lines associated to the header will be also be changed. On the other hand, if the operation is defined at the line level, only the specified line will be changed and its associated header will not be affected.
