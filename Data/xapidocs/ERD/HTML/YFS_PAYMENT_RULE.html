Entity Definition
YFS_PAYMENT_RULE
( CONFIGURATION Table )
This table stores all payment rules used by various organizations in the system.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
PAYMENT_RULE_KEYChar (24)The primary key for the YFS_PAYMENT_RULE table.  
PAYMENT_RULE_IDVarchar (20)The unique identifier for the payment rule. 
DESCRIPTIONVarchar (100)Description of the Payment Rule. 
INTERFACE_TIMEVarchar (40)This field indicates when the invoice will be published. The valid values are:
  • AT CREATION: Indicates that the invoice will be published at invoice creation time.
  • AT COLLECTION: Indicates that the invoice will be published at collection time, if the COLLECT_EXTERNAL_THROUGH_AR flag is set to 'N'. 
COLLECT_EXTERNAL_THROUGH_ARChar (1)This flag indicates whether the collection is done externally through account receivable. 
MERCHANT_IDVarchar (40)The merchant identifier used for the collection. 
FINANCIAL_HANDLING_REQUIREDChar (1)This flag indicates whether financial handling is required. 
DEF_CREDIT_ON_RET_REQChar (1)Indicates if deferred credit is required for returns held against book. 
ALLOW_REFUND_FROM_HOLD_AMOUNTChar (1)Indicates if hold amount should be considered for refunding. 
ORGANIZATION_CODEChar (24)The organization that is associated with this payment rule.
A logical foreign key to the following table:
  • YFS_ORGANIZATION 
DEFAULT_FLAGChar (1)This flag indicates whether a payment rule is the default rule for an organization. 
CUSTOMER_ACCOUNT_MAINTAINEDChar (1)This flag indicates if customer accounts spending limit is established and are maintained internally. The default value is N which means customer accounts are not setup. 
AUTH_REQUIRED_BEFORE_SCHEDULEChar (1)This flag enables or disables the requirement of acquiring an authorization before scheduling an order. 
POSTPONE_REAUTHORIZATIONChar (1)This flag enables delaying re-authorization by a number of hours, configured separately for product and service lines. 
REAUTH_HOURS_BEFORE_RELEASENumber (14,4)The number of hours before the release date to begin acquiring authorizations again for Product lines. 
REAUTH_HOURS_BEFORE_APPTNumber (14,4)The number of hours before the Promised Appointment Start Date to begin acquiring authorizations again for Service lines. 
EXTERNALLY_REQUESTED_AUTHChar (1)Identifier which prevents the normal authorization logic and bases auths on YFS_CHARGE_TRAN_REQUESTs. 
IGNORE_STATUS_FOR_PURGEChar (1)If Y, allows the order to be purged regardless of the PaymentStatus on the order. This flag is only read by the purge agent if Settlement is not required.

MODIFYUSERIDVarchar (40)User who modified the record. 
CREATEPROGIDVarchar (40)Program that created the record. 
MODIFYPROGIDVarchar (40)Program that modified the record. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
