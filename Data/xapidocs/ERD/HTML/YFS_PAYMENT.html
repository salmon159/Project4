Entity Definition
YFS_PAYMENT
( TRANSACTION Table )
This table stores details about the customer's payment methods. The key to this table is posted in YFS_CHARGE_TRANSACTION table.

Records purged from this table are archived into YFS_PAYMENT_H

You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

If the value is B then both charge and refund are suspended and all open AUTHORIZATION or CHARGE records are closed. Default: N. Valid Values: Y, B, or N.

Please note the flag comes into action only when SuspendAnyMoreCharges is either set to Y or B. Valid Values are N or Y (Null value behaves similar to setting it to N ).
