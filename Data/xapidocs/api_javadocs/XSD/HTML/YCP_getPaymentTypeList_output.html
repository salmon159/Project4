PaymentType Associated With Table: YFS_PAYMENT_TYPE
This table stores all payment types supported by various organizations.
AttributeTypeRequiredTable.Column
AllowAuthorizationsToExceedSettlementRequestflag (maximum length: 1) yfs_payment_type.allow_auth_to_exceed_settle_rq
This rule determines whether authorizations larger than the settlement amount are able to be used for settlement for this payment type. If IS_MULTIPLE_REQ_ALLOWED or CHARGE_CONSOL_ALLOWED are Y, this is implicitly Y.
AuthorizationExpirationHoursnonNegativeInteger (total number of digits: 7) yfs_payment_type.auth_expiration_hours
Number of hours an authorization is valid before expiring for this payment type. This value is not used internally by the system, but may be referenced for the computation of the AuthorizationExpirationDate when processing payment transactions.
AuthorizationReversalHoursnonNegativeInteger (total number of digits: 7) yfs_payment_type.auth_reversal_hours
Number of hours an authorization is valid for reversal for this payment type. This value is not used internally by the system and serves as an informational field.
AuthorizationReversalStrategystring (maximum length: 20) yfs_payment_type.auth_reversal_strategy
This field controls when reverse authorization requests will be created for this payment type.
Allowed ValueDescription
NO_REVERSE This strategy will never generate reversal requests for authorizations. This is the default strategy. 
EXPIRED This strategy will generate a reversal request for an authorization after it expires. 
REVERSE_EXCESS This strategy will still reverse authorizations that are expired, but it will also reverse authorizations that are in excess of either the order total or the CTR they are mapped to. In additon, any authorization that cannot be mapped to a CTR and is marked as unusable will also be reversed.

ChargeConsolAllowedflag (maximum length: 1) yfs_payment_type.charge_consol_allowed
This flag indicates whether charge consolidation is allowed for a payment type.
ChargeInsteadOfAuthflag (maximum length: 1) yfs_payment_type.charge_instead_of_auth
This flag indicates if the requestCollection API should create a CHARGE request against this Payment Type in all scenarios where an AUTHORIZATION request would be created.
ChargeSequencenonNegativeInteger (total number of digits: 5) yfs_payment_type.charge_sequence
The sequence in which charges needs to be collected if multiple payment types are supported by an organization.
ChargeUpToAvailableflag (maximum length: 1) yfs_payment_type.charge_up_to_available
This flag indicates whether the user exit getFundsAvailable should be invoked to determine the funds available on this payment account.
DefaultForReturnflag (maximum length: 1) yfs_payment_type.default_for_return
This flag indicates the usage of a payment type as the default one while processing returned goods.
HasAltRefundConstraintflag (maximum length: 1) yfs_payment_type.has_alt_refund_constraint
This flag indicates whether this payment type has some alternate refund payment type to use in case the refund amount meets the constraint.
MerchantIdstring (maximum length: 40) yfs_payment_type.merchant_id
The identifier of the merchant used for the collection.
MinWaitTimeForConsoldecimal (total number of digits: 14
fraction digits: 4) 		yfs_payment_type.min_wait_time_for_consol
The minimum time to wait for utilizing the opportunity to consolidate charges.
NewRefundPaymentTypestring (maximum length: 20) yfs_payment_type.new_refund_payment_type
The payment type that needs to be used for refunds and is only set if not refunding to the same account. Payment type selection is valid only if it belongs to payment type group STORED_VALUE_CARD or OTHER.
NoProcessingRequiredflag (maximum length: 1) yfs_payment_type.no_processing_required
This flag indicates that when the Payment Type is added on the Order, it is already processed. Examples of this Payment Type are checks/cash. A pre processed Charge Transaction record of type CHARGE is created for this Payment Type.
OrganizationCodestring (maximum length: 24) yfs_payment_type.organization_code
The organization that is associated with this payment type.
PartialReversalSupportedflag (maximum length: 1) yfs_payment_type.partial_reversal_support
This rule determines whether reverse authorization requests support partial reversal for this payment type. If Y, partial reversal is supported. If N, only full reversal is supported.
PaymentTemplateTypestring (maximum length: 100) yfs_payment_type.payment_template_type
This field indicates User Interfaces to use the template required to capture Payment Methods of this Payment Type.
PaymentTypestring (maximum length: 20) yfs_payment_type.payment_type
The unique identifier for the payment type.
PaymentTypeDescriptionstring (maximum length: 100) yfs_payment_type.payment_type_description
The description of the payment type.
PaymentTypeGroupstring (maximum length: 40) yfs_payment_type.payment_type_group
This filed indicates the payment type group, which will determine behavior.
Allowed ValueDescription
CREDIT_CARD Indicates that the payment type is in the credit card group. 
CUSTOMER_ACCOUNT Indicates that the payment type is in the customer account group. 
STORED_VALUE_CARD Indicates that the payment type is in the stored value card group. 
OTHER Indicates that the payment type is in the other group. Payment types such as checks can be configured in this group. 
DEBIT_CARD Indicates that the payment type is in the debit card group. 

PaymentTypeKeykey (maximum length: 24) yfs_payment_type.payment_type_key
The primary key for the YFS_PAYMENT_TYPE table.
RefundConstraintOperatorstring (maximum length: 4) yfs_payment_type.refund_constraint_operator
The operator to be used in evaluating the refund amount against the Refund Constraint Value.
Allowed ValueDescription
GT Indicates that the operator is greater than. 
LT Indicates that the operator is less than. 

RefundConstraintPaymentTypestring (maximum length: 20) yfs_payment_type.refund_constraint_payment_type
The payment type that needs to be used instead if the refund amount meets the constraint formed by the refund constraint operator and value when a new payment is created for refund purposes. The payment type selection is valid only if it belongs to payment type group STORED_VALUE_CARD or OTHER.
RefundConstraintValuedecimal (minimum value: 0
total number of digits: 15

RefundSameAccountflag (maximum length: 1) yfs_payment_type.refund_same_account
This flag indicates whether refunds should be issued to the same account on which the payment was made.
RefundSequencenonNegativeInteger (total number of digits: 5) yfs_payment_type.refund_sequence
The sequence in which charges need to to be refunded if multiple payment types are supported by an organization.
ReverseUnusedAuthWhenSuspendedflag (maximum length: 1) yfs_payment_type.rev_auth_when_suspend
Reverse unused authorizations when the payment method is suspended. Currently only valid for payment method of group type CUSTOMER_ACCOUNT.
ValidForReturnflag (maximum length: 1) yfs_payment_type.valid_for_return
This flag indicates whether this payment type is valid for returns.
Voidableflag (maximum length: 1) yfs_payment_type.voidable
This flag indicates that the Payment Type allows a void operation.
ZeroAmountAuthorizationSupportedflag (maximum length: 1) yfs_payment_type.zero_amount_auth_supported
This flag indicates whether or not the system will attempt a zero authorization when this Payment Type is added on the Order.
