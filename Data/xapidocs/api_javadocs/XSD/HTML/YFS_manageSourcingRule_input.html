AdditionalData
Additional flags and information to use when sourcing.
AttributeTypeRequiredTable.Column
AvailableCapacityPercentagedecimal (total number of digits: 5
fraction digits: 2) 		.
Indicates the available inventory capacity percentage that a node must have to be considered for sourcing. If not specified, then capacity percentage will not be considered in node determination.

DaysToConsiderAvailableCapacityPercentagenonNegativeInteger (total number of digits: 5) .
Number of days to consider the node's available inventory capacity for sourcing. If not specified, it will default to FutureInventoryWindow.

ExpandToMinimizeNumOfShipmentsflag (maximum length: 1) .
This flag is applicable only when optimization type is cost based, and minimizing the number of shipments. It is not applicable for Procurement Rule.
If set to Y, then the sourcing rule will be expanded further to minimize number of shipments for lines. When one of the lines for which supply is unavailable, expansion of sourcing rule is done. Based on the flag, expansion of sourcing rule will be done for lines that have supply available also, in order to minimize the number of shipments. Default value is N.

ExternallyDefinedReasonCodestring.
This attribute is only applicable for TemplateType as Externally_Defined. It can be used for custom logic in the OMPGetSourcedFromNodesExternallyUE user exit.

IdleAssetflag (maximum length: 1) .
If set to Y, indicates that only nodes where the item being sourced is an idle asset should be considered. Default value is N.

IsSmartSourcingAllowedflag (maximum length: 1) .
Determines whether this sourcing rule detail should allow for smart sourcing logic to be applied. Default value is Y.
Radiusdecimal (total number of digits: 14
fraction digits: 4) 		.
Indicates the radius from the ship to address that a node must fall within to be considered for sourcing. In the case of procurement sourcing rule, the radius from the final ship node is used. If Radius is passed, RadiusUOM must also be passed.

RadiusUOMstring (maximum length: 40) yfs_uom.uom
The unit of measure that the Radius is specified in. This should be a dimension UOM, and it must be possible to convert between this UOM and MILE.
