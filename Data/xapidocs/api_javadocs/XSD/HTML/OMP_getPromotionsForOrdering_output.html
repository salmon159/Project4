PricingRule Associated With Table: YPM_PRICING_RULE
This table provides the information for each pricing rule.
AttributeTypeRequiredTable.Column
Currencystring (maximum length: 20) ypm_pricing_rule.currency
Currency in which this rule is applicable.
Descriptionstring (maximum length: 2000) ypm_pricing_rule.description
Description of the rule.
EnableMixNMatchItemsflag (maximum length: 1) ypm_pricing_rule.enable_mix_n_match_items

Indicates that discount can be applied in the scenario where the combinations can not satisfied by single item, but can be satisfied by including Items which belongs to same category or having same attribute values defined in the rule. Default value is null.
When this flag is enabled for Quantity rules and WHEN_TO_APPLY=SINGLE_ITEM, then field TargetItemDeterminationRule value is used for selecting to which item adjustment should be applied. If the TargetItemDeterminationRule value is null, this value will be defaulted to HIGH_PRICED_ITEM. This flag is not applicable for getItemPrice API. Note: Item aggregated quantity will not be distributed among the rule's criteria to satisfy the criteria condition.

EndTimetimeypm_pricing_rule.end_time

ExclusivityIdstring (maximum length: 100) ypm_pricing_rule.exclusivity_id
ExclusivityID is used to group set of Pricing Rules for mutual exclusivity. Exclusivity Type settings are applicable among Pricing Rules having same ExclusvityID
ExclusivityTypestring (maximum length: 40) ypm_pricing_rule.exclusivity_type
Exclusivity Type of the Pricing Rule
Allowed ValueDescription
ALL Can combine with any other pricing Rule. (default value) 
RULE_TYPE_ITEM Cannot combine with Pricing Rules of same RuleType for same Item. 
RULE_TYPE_ITEM_INST Cannot combine with Pricing Rules of same RuleType for same Item instance. 
RULE_TYPE_ORDER Cannot combine with Pricing Rules of same RuleType for same Order. 
ONE_RULE_WITHIN_ID Cannot combine with any other Pricing rule within the same id 
GLOBAL Cannot combine with any other Pricing rule

PrioritynonNegativeInteger (total number of digits: 10) ypm_pricing_rule.priority
This attribute is used to prioritize Pricing Rule among a list of applicable pricing rules with same Exclusivity settings. Higher the value of this attribute, higher the priority of Pricing Rule, to be applied to an order/item.
RuleCategorystring (maximum length: 40) ypm_pricing_rule.rule_category
Indicates the category to which the rule belongs PRICING or SHIPPING.
Allowed ValueDescription
PRICING Indicates that this rule belongs to pricing category. This rule generates adjustments for extended price of a line or for order total before shipping and tax. 
SHIPPING Indicates that this rule belongs to shipping category. This rule generates shipping surcharge for a line or adjustment on shipping charge of the order. 

RuleTypestring (maximum length: 40) ypm_pricing_rule.rule_type
Indicates the type of the rule COMBINATION ORDER_TOTAL ITEM_QUANTITY or SURCHARGE
Allowed ValueDescription
COMBINATION Indicates that this rule is applied based on combination of items in the order. 
ORDER_TOTAL Indicates that this rule is applied based on order total before shipping or tax. 
ITEM_QUANTITY Indicates that this rule is applied based on item and it's quantity in the order. 
SURCHARGE Indicates that the rule is for applying a surcharge. 
GLOBAL_EXCLUSION Indicates that the rule is for excluding certain items, categories or attributes from all pricing rules.

PricingRuleAction Associated With Table: YPM_PRICING_RULE_ACTION
This table maintains the information about the actions of a pricing rule. One of the actions is fired depending on the condition satisfied. This table also maintains the tiered conditions of order total and item quantity rule.
AttributeTypeRequiredTable.Column
Adjustmentdecimal (total number of digits: 19
fraction digits: 6) 		ypm_pricing_rule_action.adjustment
Adjustment to be applied as part of this action.
AdjustmentTypestring (maximum length: 40) ypm_pricing_rule_action.adjustment_type
Type of adjustment. FLAT_FEE, PERCENT or ABSOLUTE.
Allowed ValueDescription
FLAT_FEE Applicable only for shipping order total rule. A flat fee action generates an adjustment such that the shipping charge is discounted to the value specified in the adjustment. 
PERCENT Indicates that the adjustment is a percentage on the unit price of an item or order total before shipping and tax or on the shipping charge of an item for shipping rules. 
ABSOLUTE Indicates that the adjustment is a absolute value on the unit price of an item or order total before shipping and tax for pricing rules or on the shipping charge of an item for shipping rules. 
FIXED_LINE_PRICE Indicates the rule is for fixed cost for a particular item or combination of multiple items. 
ABSOLUTE_TOTAL Applicable only for shipping item quantity rule. Indicates that the adjustment is a absolute value on the total shipping charge across all applicable items. 
FLAT_FEE_TOTAL Applicable only for shipping item quantity rule. A flat fee total action generates an adjustment such that the total shipping charge across all applicable items is discounted to the value specified in the adjustment. 

ApplyAdjustmentToRemainderflag (maximum length: 1) ypm_pricing_rule_action.apply_adj_to_remainder
Indicates that adjustments should be applied only to the remainder of the transaction.
ChargeCategorystring (maximum length: 20) ypm_pricing_rule_action.charge_category
Unique identifier of the Charge Name. Category to which the charge name belongs to.
MaxAbsoluteAdjustmentdecimal (total number of digits: 19
fraction digits: 6) 		ypm_pricing_rule_action.max_absolute_adjustment
Maximum absolute adjustment that can be provided for this pricing rule action within a transaction.
PricingRuleActionKeykey (maximum length: 24) ypm_pricing_rule_action.pricing_rule_action_key
Primary key attribute of YPM_PRICING_RULE_ACTION table.
PricingRuleKeykey (maximum length: 24) ypm_pricing_rule_action.pricing_rule_key
The rule this action is associated.
QualifierAmountdecimal (total number of digits: 19
fraction digits: 6) 		ypm_pricing_rule_action.qualifier_amount
A number that needs to be satisfied in conjunction with the QUALIFIER_AMOUNT_TYPE for the rule condition to satisfy.
QualifierAmountTypestring (maximum length: 40) ypm_pricing_rule_action.qualifier_amount_type
Specifies qualifier amount type of the action. Qualifier amount type can be based on minimum unit price or minimum total extended price.
Allowed ValueDescription
TRIGGER_MINIMUM_UNIT_PRICE This rule condition satisfies only when trigger item's unit price is greater or equal to the value in the QUALIFIER_AMOUNT column. 
TRIGGER_MINIMUM_TOTAL_EXTENDED_PRICE This rule condition satisfies only when the sum of trigger items' extended prices is greater or equal to the value in the QUALIFIER_AMOUNT column. 
TRIGGER_MINIMUM_CUMULATIVE_EXT_PRICE This rule condition satisfies only when the sum of the trigger items' extended prices plus any already applied discounts to the trigger items is greater or equal to the value in the QUALIFIER_AMOUNT column. 
TARGET_MINIMUM_UNIT_PRICE This rule condition satisfies only when the target item's unit price is greater or equal to the value in QUALIFIER_AMOUNT column. 
TARGET_MINIMUM_TOTAL_EXTENDED_PRICE This rule condition satisfies only when the sum of target items' extended prices is greater or equal to the value in the QUALIFIER_AMOUNT column. 
TARGET_MINIMUM_CUMULATIVE_EXT_PRICE This rule condition satisfies only when the sum of the target items' extended prices plus any already applied discounts to the target items is greater or equal to the value in the QUALIFIER_AMOUNT column. 

Quantitydecimal (minimum value: 0
total number of digits: 14
fraction digits: 4) 		ypm_pricing_rule_action.quantity
Quantity of triggering item for tiered actions. Applicable only for rule type item quantity.
TargetItemDeterminationRulestring (maximum length: 20) ypm_pricing_rule_action.target_item_deter_rule
Specifies on which target item the action should be applied. This is required while specifying apply on: 'one quantity of target item' or 'every combination'
Allowed ValueDescription
LOW_PRICED_ITEM Indicates that the adjustment is to be applied on item with the lowest unit price. 
HIGH_PRICED_ITEM Indicates that the adjustment is to be applied on item with the highest unit price. 
SAME_LOW_PRICE_ITEM Indicates that the adjustment is to be applied on item with the same or less unit price than any item on which the pricing rule is triggered. 
ALL_ITEMS This value is used when defining combination rules with target as 'All items in order', but excludes certain items. 

TargetOccurrenceLimitstring.
Maximum Number of occurrences of adjustment because of this rule in an order.
WhenToApplystring (maximum length: 40) ypm_pricing_rule_action.when_to_apply
Indicates how to apply the action. As SINGLE_ITEM, ALL or EACH_COMBINATION.
Allowed ValueDescription
SINGLE_ITEM Indicates that the rule adjustment is applicable on one unit of target item. 
ALL Indicates that the rule adjustment is applicable on all units of target item. 
EACH_COMBINATION Indicates that the rule adjustment is applicable on the target item, as many times as the rule condition is satisfied. Applicable only for combination type of rules. 
SPECIFIC_NUMBER Indicates that the rule adjustment is applicable on specific units of target item
