ExecutionException Associated With Table: YFS_EXECUTION_EXCEPTION
This table stores the user-defined execution exception codes.
AttributeTypeRequiredTable.Column
AllowPartialDepositflag (maximum length: 1) yfs_execution_exception.allow_partial_deposit
This flag indicates whether to allow partial deposit of picked inventory at the suggested location. Valid values are Y and N. Default value is N.
CheckDigitstring (maximum length: 5) yfs_execution_exception.check_digit
Check Digit is a numeric representation of Execution Exception. It is generally used by Voice Systems to verify the Execution Exception. Execution Exception Check Digit is unique for a Node.
ExceptionCodestring (maximum length: 40) yfs_execution_exception.exception_code
This field specifies the unique code of the user defined execution exception.
ExceptionLongDescriptionstring (maximum length: 255) yfs_execution_exception.exception_long_description
This field specifies the long description of the user defined execution exception.
ExceptionShortDescriptionstring (maximum length: 40) yfs_execution_exception.exception_short_description
This field specifies the short description of the user defined execution exception.
ExceptionTypestring (maximum length: 40) yfs_execution_exception.exception_type
This field indicates the execution exception types. Valid values are: PICK, DEPOSIT, and SKIP.
Allowed ValueDescription
PICK Pick Execution Exception 
DEPOSIT Deposit Execution Exception 
SKIP Skip Execution Exception 

ExecutionExceptionKeykey (maximum length: 24) yfs_execution_exception.execution_exception_key
The primary key for the YFS_EXECUTION_EXCEPTION table.
FlowKeykey (maximum length: 24) yfs_execution_exception.flow_key
The flow key that is associated with this execution exception.
FreezeSourceForMoveOutflag (maximum length: 1) yfs_execution_exception.freeze_source_for_move_out
This flag indicates whether the source location is frozen for moving out the inventory.
FreezeTargetForMoveInflag (maximum length: 1) yfs_execution_exception.freeze_target_for_move_in
This flag indicates whether the target location is frozen for moving in the inventory.
HoldTaskflag (maximum length: 1) yfs_execution_exception.hold_task
This flag indicates whether the task should be held.
InventoryStatusForUnavailableQtystring (maximum length: 40) yfs_execution_exception.inv_sts_for_unavailable_qty
This field indicates the inventory status of the unavailable item quantity.
MarkTargetAsFullflag (maximum length: 1) yfs_execution_exception.mark_target_as_full
This flag indicates whether the target location should be marked as full.
MaximumRecordsnonNegativeInteger (total number of digits: 5) .
Specifies maximum number of records that must be fetched.
Nodekey (maximum length: 24) yfs_execution_exception.node_key
The node key that is associated with this execution exception.
This attribute corresponds to the OrganizationCode Fact.
SuggestedAlternateDepositLocationflag (maximum length: 1) yfs_execution_exception.suggested_alt_deposit_lcn
This flag indicates whether an alternate deposit location should be suggested. This is used when a deposit exception is encountered.
SuggestedAlternatePickLocationflag (maximum length: 1) yfs_execution_exception.suggested_alternate_pick_lcn
This flag indicates whether an alternate pick location is suggested. This is used when a pick exception encounters.
UnassignUserflag (maximum length: 1) yfs_execution_exception.unassign_user
This flag indicates whether the current task should be rejected.
Flow Associated With Table: YFS_FLOW
This table stores all the configured services. It is populated when configuring services using the service builder framework or when configuring time triggered transactions for agents.
AttributeTypeRequiredTable.Column
FlowGroupNamestring (maximum length: 40) yfs_flow.flow_group_name
The service group name, which is used as a logical placeholder for the services.
FlowKeykey (maximum length: 24) yfs_flow.flow_key
Primary key attribute of YFS_FLOW.
FlowNamestring (maximum length: 128) yfs_flow.flow_name
The flow name is a unique identifier of the service. The field size is 128 bits to accommodate the backward compatibility naming format.
FlowTypestring (maximum length: 2) yfs_flow.flow_type
The flow type identifies the service associated with this table.
Allowed ValueDescription
00 Indicates that this service belongs to an Integration server. This is the default value. 
01 Indicates that this service belongs to an AgentServer. 

GraphUIKeykey (maximum length: 24) yfs_flow.graph_ui_key
This identifier is a reference to the YFS_GRAPH_UI table, where the pictorial representation of the service is stored.
IsOutGoingflag (maximum length: 1) yfs_flow.is_out_going
This flag indicates whether the service is invoked synchronously or asynchronously.
IsRealTimestring (maximum length: 1) yfs_flow.is_real_time
This flag determines whether a real time response is needed.
Allowed ValueDescription
Y Indicates a real time response is needed. In this case, an asynchronous transport node is not allowed in the service. 
N Indicates a real time response is not needed. 

OwnerKeykey (maximum length: 24) yfs_flow.owner_key
This key indicates the owner of the service. This value is "DEFAULT", which is the hub organization for all services. However, when moving event handlers and user exits, the user interface populates this with the appropriate enterprise organization key.
This attribute corresponds to the OrganizationCode Fact.
ProcessTypeKeykey (maximum length: 24) yfs_flow.process_type_key
This key determines all the services belonging to some process type
TransportTypeKeykey (maximum length: 24) yfs_flow.transport_type_key
The transport type key of the beginning transport node for asynchronous services or LOCAL for synchronous services.
