OrderLine Associated With Table: YFS_ORDER_LINE
The line which is getting split. Must pass a unique identifier; either OrderLineKey or both PrimeLineNo and SubLineNo. No other attributes can be modified using splitLine.
If the input XML contains references to multiple OrderLine elements, and some of those references are to the same OrderLine, splitOrder does not throw an error. splitOrder processes only the last of the OrderLines, along with its children nodes. All the preceding OrderLine which refer to the same line be ignored.
An OrderLine can be split into only one new line. To break an existing OrderLine into multiple new lines, splitOrder must be called multiple times.

Validation ConditionError
The OrderLineKey attribute OR the PrimeLineNo and SubLineNo attributes must be passed and must be valid.

The line which is getting split cannot be a bundle parent line.

SplitLine Associated With Table: YFS_ORDER_LINE
The OrderLine which is going to be added. Information is taken from the existing, or source, line and used to populate the new OrderLine, based on appropriate templates.
Template: Two different templates are provided for each document type. Configure the templates depending on what needs to be copied to the new line.
SplitLineDefaultTemplate: Used when line is not split based on Item. All the line attributes can be configured to copy into the new line in this case.
SplitLineItemSubstitutionTemplate: Used when line is split based on Item. If any of the item attributes ItemID, ProductClass, UnitOfMeasure are different for the new line from the original line, then this template is used. This template can not have AllocationDate, KitCode, Item Attributes or KitLines. These attributes are not copied to new line if they exist on the original line.
If template allows OrderLineOption and its CustomAttributes, they are copied to new order line.

Defaulting of line references is not supported for the SplitLine. The SplitLineDefaultTemplate cannot have Reference element.

Attributes LineSeqNo and OriginalOrderLineKey are always copied to the new line regardless of the template values.

If the attributes AdditionalAddresses or Instructions are passed in override xml and are contained in the SplitLine template also, then new line information is from both the parent line and the new information passed.

Other attributes work in replace mode. If passed in override xml, the new line will use that information.

When ReservationID is passed, ToStatus must not be passed. The splitline gets created into system determined status.

The split into line is validated like any other new line. Refer to changeOrder element /Order/OrderLines/OrderLine for more details on attributes validations performed for this element.
