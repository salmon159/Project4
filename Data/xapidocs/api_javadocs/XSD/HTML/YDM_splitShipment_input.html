ShipmentLine Associated With Table: YFS_SHIPMENT_LINE
To identify the ShipmentLine, pass either ShipmentLineKey or ShipmentLineNo and ShipmentSubLineNo or Order attributes.
If ShipmentLine is a logical kit parent line then kit component lines also will be considered in the proper kit ratio.
If ShipmentLine is a bundle parent line then the bundle component lines that are not specified for splitting will be moved to the target shipment. The quantity moved from bundle components is minimum of (Quantity Available to Split, KitQty * Parent Quantity Specified).
If ShipmentLine is a bundle component line then the bundle parent lines up to the bundle root will be pulled to the target shipment.

ShipmentTagSerial Associated With Table: YFS_SHIPMENT_TAG_SERIAL
Additional information related to tag number, serial number, ship by date for a shipment line is stored in this table. If the shipment line item is tagged, ShipmentTagSerial has to be specified to move the tagged shipment line quantity. To identify the ShipmentTagSerial pass ShipmentTagSerialKey, or SerialNo, or ShipByDate or Tag Attributes: LotNumber, LotKeyReference, ManufacturingDate, LotAttribute1, LotAttribute2, LotAttribute3, BatchNo, RevisionNo. Tag information is stored only if the Node tracks tag information in inventory or the Buyer mandates tag information.

Validation ConditionError
When sum of tagged quantity of the line and containerized quantity is greater than ShipmentLine quantity.
