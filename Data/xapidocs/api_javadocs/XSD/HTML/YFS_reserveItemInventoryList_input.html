ExpirationDatedateTimeyfs_inventory_reservation.expiration_date
Expiration date for the reservation. If not pass and the DEFAULT_RESERVATION_EXPIRATION_TIME rule is setup with a value greater than 0, the expiration date will be calculated as ship date + DEFAULT_RESERVATION_EXPIRATION_TIME.
Default: High Date.
If more quantities are added to an existing reservation, the ExpirationDate will be recalculated.

With Inventory Visibility (IV) integration enabled,
this ExpirationDate will be converted into minutes beyond the current time before calling IV reservations API. The number of minutes will be passed as reservation expiration duration to the IV API. If the value of ExpirationDate is found to be older than the current time then an error with code IVADP0001 is thrown.
If no ExpirationDate is passed and the DEFAULT_RESERVATION_EXPIRATION_TIME rule is setup with a value greater than 0, then the same will be converted into minutes and passed as reservation expiration duration to the IV reservation API. If the rule is not setup or does not have a value greater than 0, then a default reservation expiration duration of 120 minutes is passed to the IV API.

ReservationIDstring (maximum length: 100) Yesyfs_inventory_reservation.reservation_id
Reservation is created for the passed value for ReservationID. The same ReservationID must be passed at the time of creating order if the reservation has to be consumed.

With Inventory Visibility (IV) integration enabled, for creating new reservations in IV, this ReservationID will be passed as reference for calling the IV reservation API.
For cancelling existing reservations, this ReservationId can be either of the following
1. Reservation Id of the IV reservation.
In this case, the reservation record corresponding to the given IV reservation Id will be fetched. The other input parameters for this api will not be used to match the reservation record obtained from IV because IV reservation Id can be used to uniquely identify a reservation record in IV. If a reservation record is found, it will be cancelled by calling IV reservation API.
2. Reservation reference of the IV reservation.
In this case, the reservation record(s) corresponding to the given IV reservation reference will be fetched. There may be multiple reservation records corresponding to a given IV reference hence the input attributes ItemId, ShipNode, ProductClass, UnitOfMeasure, ReservationPool and ReservationPoolType will be used to match against the IV reservation records. The matched reservation records will be cancelled by calling IV reservation API.
