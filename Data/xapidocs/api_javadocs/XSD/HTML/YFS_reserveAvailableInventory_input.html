With Inventory Visibility (IV) integration enabled,
this ExpirationDate will be converted into minutes beyond the current time before calling IV reservations API. The number of minutes will be passed as reservation expiration duration to the IV API. If the value of ExpirationDate is found to be older than the current time then an error with code IVADP0001 is thrown.
If no ExpirationDate is passed and the DEFAULT_RESERVATION_EXPIRATION_TIME rule is setup with a value greater than 0, then the same will be converted into minutes and passed as reservation expiration duration to the IV reservation API. If the rule is not setup or does not have a value greater than 0, then a default reservation expiration duration of 120 minutes is passed to the IV API.
