AdjustmentTypestring (maximum length: 15) Yesyfs_inventory_audit.transaction_type
The type of adjustment.
Valid Values:
ADJUSTMENT - indicates manual inventory adjustment
RECEIPT - indicates a receipt of a PO or ASN.
ABSOLUTE - indicates the absolute inventory quantity adjustment. For Absolute AdjustInventory calls, the input is only expecting one update per supply record. When multiple absolute adjustments are made to the same supply record, the behavior is undefined.The ABSOLUTE adjustment type is not meant for supplies with INFINITE availability type.

Validation ConditionError
Must be a valid value.
YFS10385Invalid Adjustment Type

Availabilitystring (maximum length: 20) yfs_inventory_supply.availability_type
Defines whether the supply is infinite or tracked.
Valid values: TRACK or INFINITE
Default: If not passed then it defaults to Availabilty type of the shipnode. If ShipNode tracks inventory it will default to TRACK else it will default to INFINITE
The behavior depends on the current configuration of the ShipNode and the value passed in Availability.

ShipNode Configuration Availability passed API Behavior
TRACK TRACK Updated based on quantity passed
TRACK INFINITE Supply record stamped as infinite
INFINITE TRACK Updated based on quantity passed
INFINITE INFINITE No updates done

Validation ConditionError
Input must either be TRACK or INFINITE.
YFS10190Invalid Data.

Allowed ValueDescription
INFINITE Indicates that availability is infinite. 
TRACK Indicates that availability is tracked.

FromETAdateyfs_inventory_supply.eta
Used when the AdjustmentType value is RECEIPT , indicating a receipt of a PO and ASN into the ShipNode. If FromETA is supplied, the Quantity is subtracted from the supply record that matches FromSupplyType, (FromLotNumber or information in FromTag element), FromShipByDate, FromSegment, FromSegmentType, FromSupplyReference, FromSupplyReferenceType and FromETA. When SupplyType is of the type onhand, FromETA is not required. If FromSupplyType is not of the type onhand, FromETA is mandatory.

FromShipByDatedateyfs_inventory_supply.ship_by_date
Used when the AdjustmentType value is RECEIPT , indicating a receipt of a PO and ASN into the ShipNode. If FromShipByDate is supplied, the Quantity is subtracted from the supply record that matches FromSupplyType, (FromLotNumber or information in FromTag element), FromShipByDate, FromSegment, FromSegmentType, FromSupplyReference, FromSupplyReferenceType and FromETA. If the Item is time-sensitive, a ShipByDate may be provided. If ShipByDate is not provided, then if
  • SupplyType is of the type onhand the ShipByDate is calculated as "today + item's default expiration days".
  • SupplyType is not of the type onhand theShipByDate is calculated as "ETA + item's default expiration days".
If ShipByDate is provided, then it is used for updates.

Validation ConditionError
