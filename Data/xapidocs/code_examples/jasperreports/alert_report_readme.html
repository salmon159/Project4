Section 2.0 - Installation Procedure

    This sample was compiled with Jasper Reports version 3.6.1. If you wish to run a different version of Jasper Reports, you must download dependent jars compatible with the version you wish to use. You must also recompile the sample reports with your version.

    This sample takes an Alert and generates a report from it. It is a sample of using the Jasper Reports library within a JSP page, without using the SDF component.

  1. Download the following file:
    • jasperreports-3.6.1-project.zip from http://jasperforge.org/projects/jasperreports/
    This file contains the JasperReports jar and its dependencies. Alternatively, the dependencies may be downloaded independently from their respective sites.

  2. Extract the following jar files from the zip file downloaded in Step (1):
    • dist/jasperreports-3.6.1.jar
    • lib/commons-beanutils-1.8.0.jar
    • lib/commons-collections-2.1.1.jar
    • lib/commons-digester-1.7.jar
    • lib/commons-logging-1.0.4.jar
    • lib/iText-2.1.0.jar

  3. Run install3rdParty.sh on all the jars specified in Step (1) and Step (2) to add them to the "APP" classpath

  4. Create a directory called JasperReports

  5. Copy the compiled version of the sample report files (sampleAlertAuditReport.jasper, sampleAlertCommentsReport.jasper, sampleAlertReferenceReport.jasper, and sampleAlertReport.jasper) into the JasperReports directory. Remember these must be compiled with the exact version of JasperReports in use.

  6. Copy the samplealertreport.properties file into the JasperReports directory.

  7. Copy the testAlertPrint.jsp into the [INSTALL]/repository/eardata/[APPLICATION]/war/extn directory.

  8. Run alertyfsresource.sql to load the Resource and Resource Permission records into the database. This adds an action (Print_Report) in the Alert console list screen.

  9. Create a directory called com/yantra/jasperreports under the JasperReports directory.

  10. Compile the SampleAlertAuditReportScriptlet.java, SampleAlertCommentsReportScriptlet.java, SampleAlertReferenceReportScriptlet.java and SampleAlertReportScriptlet.java files and copy the SampleAlertAuditReportScriptlet.class, SampleAlertCommentsReportScriptlet.class, SampleAlertReferenceReportScriptlet.class and SampleAlertReportScriptlet.class files to the JasperReports/com/yantra/jasperreports directory.
    These classes are used by the report to format dates returned by the Sterling API.

  11. Package the contents of the JasperReports directory into a jar file and run install3rdParty.sh on it.

  12. Rebuild the EAR. For more information about creating an EAR refer to the Application Installation Guide.

  13. Restart your application server or deploy the new archive onto it.

  14. Go to the Sterling applications Alert List screen. Select an Alert and choose action Print_Report.
    This should open a new window with the report rendered in PDF format. The file sampleAlertReport.pdf is a sample of what this Alert Report will look like.
